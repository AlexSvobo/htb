examining its contents, it appears to be a replica of the SYSVOL share. This is particularly interesting from a
privilege escalation standpoint, as Group Policies—and potentially Group Policy Preferences—are stored
within SYSVOL , which is readable by all authenticated users. For more details on exploiting this, refer to this
resource.

This happens because the remote Windows system is using a non-default code page (character encoding), so some characters (like accented letters or symbols) are not decoded correctly by your tool (psexec or wmiexec). That’s why you see “��” instead of proper characters.

Why is wmiexec output cleaner?

wmiexec.py uses WMI to execute commands and capture output, which often handles encoding better and avoids some of the quirks of SMB pipes used by psexec.py.
psexec.py relies on SMB and may not handle code pages or output formatting as well, especially with non-ASCII characters.
