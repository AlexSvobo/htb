How getcap exploits work:

getcap shows binaries with Linux capabilities set (like cap_setuid+ep).
cap_setuid+ep allows a binary to change its user ID to any user, including root, without being SUID.
If you can execute such a binary, you can write a script (like in perl or python) to set your UID to 0 (root) and spawn a shell.
This bypasses the need for SUID root, as the capability grants the same privilege.
