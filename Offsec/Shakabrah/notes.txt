proof.txt



 explicitly set the UID to 0, used the correct embedded Python (py3), and spawned a pty — the GTFOBins line didn’t do all three, so it often fails on modern systems.

Why the working command succeeded (what changed)

It used the Python 3 interpreter that this vim build actually includes (+python3). Many GTFOBins examples use py (python2); if vim only has python3 those will be ignored.
It called os.setuid(0) before creating a shell. That forces the process’ real/effective UID to root. Relying on exec alone (os.execl) is less reliable on modern systems where privilege semantics or the launched shell may drop privileges.
It used pty.spawn("/bin/bash"). That creates a proper pseudo‑TTY for an interactive bash, so job control and interactive features work. The GTFOBins variant spawns a plain sh without allocating a pty, so the shell is broken or unusable.
You suppressed vim config/runtime with -u NONE -N -i NONE -U NONE which removes noisy side effects (the E79 spam) and avoids unexpected plugin/rc behavior that can interfere with the exploit.
Why the GTFOBins line often fails

Wrong interpreter: GTFOBins often shows multiple variants; if you run the wrong one (py vs py3 vs perl), it simply won’t run because that feature isn’t compiled in.
No explicit setuid call: os.execl replaces the process image but doesn’t guarantee real UID is set to 0 in all cases; some shells or environments will drop privileges if the real UID isn’t root. Calling os.setuid(0) is explicit and reliable.
No pty allocation: spawning /bin/sh without a pty leaves you in a minimal, non-interactive shell (no job control, broken terminal behavior).
Quoting/command context and vim runtime files can break the payload if not suppressed.
What each part of the working one‑liner does

-u NONE -N -i NONE -U NONE : don’t load vimrc, plugins, viminfo — reduces noise and side effects.
-c 'py3 import os,pty; os.setuid(0); pty.spawn("/bin/bash")' : use vim’s python3 hook, set UID to 0, then spawn a bash attached to a new pty.
Using python3 vs python2 : matches the feature set printed by vim --version (+python3, -python).
